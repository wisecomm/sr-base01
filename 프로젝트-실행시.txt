
- 터미널에서 실행 : pnpm dev (포트 3000)
- Run Debug 실행 : 스프링

# 배포 빌드 방법
Prod 빌드 (기본값):
명령어: ./gradlew build 또는 ./gradlew build -Pprofile=prod
프론트엔드: pnpm build 실행 (기본 .env.production 사용)
Dev 빌드:
명령어: ./gradlew build -Pprofile=dev
프론트엔드: pnpm run build:dev 실행 (.env.development 사용)


. 프론트엔드 작업 시 (빠른 피드백)
UI 수정 사항을 즉시 확인하고 싶을 때 사용하는 방법입니다.

-- 프론트 실행: nextjs-client 폴더에서 pnpm dev (포트 3000)
-- 백엔드 실행: spring-rest 폴더에서 ./gradlew bootRun (포트 8080)
-- 테스트: nextjs-client 폴더에서 pnpm test:e2e (Playwright 테스트)


2. 통합 및 배포 준비 시 (통합 확인)
Next.js가 Spring Boot 내부에 포함된 상태로 잘 작동하는지 확인할 때 사용합니다.

전체 빌드: spring-rest 폴더에서 ./gradlew build
이 명령 하나로 Next.js 빌드 -> 정적 파일 복사 -> Java 빌드 -> JAR 생성이 한 번에 진행됩니다.
프론트엔드만 갱신: ./gradlew buildFrontend copyFrontend
Java 코드는 수정하지 않고 프론트엔드 화면만 반영하고 싶을 때 유용합니다.
백엔드 테스트: spring-rest 폴더에서 ./gradlew test (JUnit 테스트)
